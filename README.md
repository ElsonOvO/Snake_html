# Snake_html
我的第一个AI-Native项目：由AI生成代码、手动纠错完成的网页贪吃蛇
#介绍
  这是一个完全由ai生成的HTML/CSS/JavaScript小游戏，用于测试ai生成代码的质量和速度，以及用于学习实践最简单的ai项目如何完成，也是我的第一步。这份README更多是记录这个小项目的过程而非怎么使用，其实在这个贪吃蛇项目开始之前我就尝试了用ai做一些实用的工具比如：能快速检索文件的工具，但是最后停留在如何让它使用windows系统的api以及已经有很多同类型工具做到了；能按快捷键截屏并将图中文字识别然后翻译的工具，停留在文字识别需要用到大公司的模型api接口，但是这需要💴，并且也已经有人做出类似的软件了。然后我就一直在想有没有不依赖需要米的api接口同时足够轻量化和实用的概念可以实现，这实在不容易，市面上没有的、能解决生活中痛点、不需要其它公司接口、轻量化的加起来的要求可能确实太高了，于是就有从最简单的贪吃蛇小游戏开始做起的想法。我所用的提示词：生成一个贪吃蛇小游戏，画面精美，界面设计现代美观简洁（ps：防止ai生成像素风或命令行式的界面）；当鼠标停留在按钮上时，按钮会适度放大作出反馈；提供自定义背景图片功能（ps：如果ai的设计不符合预期至少还有自定义功能）；提供游戏内暂停功能；排行榜功能。

#操作说明
 这份html文件下载之后双击即可打开
| 操作 | 键盘/鼠标 | 触屏 |（ps：是的ai甚至做了移动端适配）
| 移动蛇 | 方向键 / WASD | 屏幕下方方向键 或 画布滑动 |
| 暂停/继续 | 空格键 / P 键 | 点击“暂停”按钮 |
| 开始游戏 | 点击“开始游戏”按钮 | 同样 |
| 返回主菜单 | ESC 键 或 点击按钮 | 同样 |
| 保存分数 | 在结束界面输入名字后点击“保存分数” | 同样 |

#过程中遇到的问题
在没有插入自定义图片时，主页竟然直接显示了一个“您的图片”；死亡停留界面的ui显示不全，“骷髅头”和“返回主页按钮”都被遮挡。将这些反馈给ai后问题都成功解决。
